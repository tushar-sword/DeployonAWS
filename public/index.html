Setup IAM User

Why IAM User? Using IAM instead of root account is a security best practice. It provides:
  • Limited permissions with guardrails
  • Better access control and audit trails
  • Reduced risk of accidental damage
  • Principle of least privilege

Create IAM User

1. Go to AWS Console ‚Üí IAM

2. Click Users ‚Üí Add User

3. Enable AWS Management Console access

4. Attach AmazonEC2FullAccess policy

5. Download credentials and save securely

6. Log out from root and log in as IAM user

Configuration Steps

1. Name your instance: my-nodejs-server

2. Choose AMI: Ubuntu Server 22.04 LTS

3. Instance Type: t2.micro (Free tier)

4. Key Pair (CRITICAL!):

Create and download demo.pem - You cannot download it again! Store it securely.

5. Network Settings:

  • ‚úÖ Allow SSH traffic from Anywhere
  • ‚úÖ Allow HTTP traffic
  • ‚úÖ Allow HTTPS traffic

4

Connect via SSH

Setup Key Permissions

Move your key file to a secure location:

mv ~/Downloads/demo.pem ~/.ssh/

Set read-only permissions (required for SSH):

chmod 400 ~/.ssh/demo.pem

Connect to Your Instance

Get your instance's Public IPv4 address from EC2 dashboard, then:

ssh -i ~/.ssh/demo.pem ubuntu@YOUR_INSTANCE_IP

Type yes when prompted to accept the connection.

Configure Security Groups

Why isn't my app accessible? By default, EC2 only exposes port 22 (SSH). You need to open your application port!

Open Application Port

1. Go to EC2 Dashboard ‚Üí Select your instance

2. Click Security tab ‚Üí Click Security Group

3. Edit inbound rules ‚Üí Add rule

4. Configure:

  • Type: Custom TCP
  • Port range: 3000
  • Source: 0.0.0.0/0

5. Save rules

Access Your Application

Open your browser and navigate to:

http://YOUR_INSTANCE_IP:3000
